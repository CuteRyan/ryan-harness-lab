# Daily Report (진행 노티)

하루 작업을 마무리할 때, 대표/팀원에게 공유할 수 있는 진행 노티를 생성하는 스킬입니다.

## Trigger
- 사용자가 "마무리하자", "진행 노티", "오늘 정리", "팀에 공유", "노티 해줘" 등을 말할 때
- `/daily-report` 명령어로 직접 호출

## How it works

1. **오늘 한 작업 파악** — 현재 대화 내용 + `docs/HISTORY.md` 최신 섹션을 참조
2. **아래 포맷으로 정리** — 팀원(비개발자 포함)이 이해할 수 있는 수준으로 작성
3. **기술 용어 최소화** — API 이름, 라이브러리명 등은 풀어서 설명하거나 괄호로 보충
4. **고민/아이디어도 포함** — 확정된 것뿐 아니라, 논의 중인 방향과 고민도 포함

## 출력 포맷

텔레그램 그룹 채팅방에 바로 붙여넣기 가능한 플레인 텍스트로 출력한다.

```
[프로젝트명] 개발 진행 공유 (YYYY-MM-DD)

[완료된 작업]
• 무엇을 했는지 (비개발자도 이해 가능하게)
• 결과가 뭔지

[인프라/환경 변경]
• 외부 서비스 연동, 비용 변화 등 (해당 시에만)

[다음 단계 방향]
• 어떤 방향으로 가고 있는지

[현재 고민/논의 중인 부분]
• 아직 결정 안 된 것, 아이디어, 의견이 필요한 것

[다음 작업]
• 구체적인 다음 작업
```

## Rules
- **마크다운 서식 절대 금지** — *, **, #, ### 등 마크다운 문법 사용 금지. 볼드/이탤릭 없이 플레인 텍스트만 사용. 섹션 구분은 [대괄호]로, 항목은 • 불릿으로
- **코드블록으로 감싸서 출력** — 사용자가 바로 복사해서 텔레그램에 붙여넣을 수 있도록
- **팀원은 비개발자 포함** — 코드 변수명, 파일 경로, 기술 내부 구현 디테일은 빼고, "어떤 기능이 어떻게 바뀌는지"에 집중
- **영어 용어는 반드시 설명 병기** — 처음 등장하는 영어 용어는 괄호로 뜻을 설명. 예: "CI(자동 테스트 시스템)", "PR(코드 변경 요청)", "DB(데이터베이스)". 이미 한 번 설명한 용어는 반복 설명 불필요
- **8단 검색 같은 내부 용어** → "인기 아티스트, 신인, 수상자, 뉴스 등 8가지 관점으로 검색" 처럼 풀어서 설명
- **분량**: 스크롤 1~2번 내로 읽을 수 있는 정도 (너무 길지 않게)
- **톤**: 명사형 종결 (예: "~수정", "~완료", "~예정"). 존댓말 아닌 간결한 명사형으로 끝냄
- **고민 섹션 필수** — 확정된 것만 나열하면 팀원이 참여할 수 없음. "이런 부분을 고민 중입니다"를 꼭 포함

## File Locations
- 참조: `{프로젝트폴더}/docs/HISTORY.md` (최신 섹션)
- 참조: `~/.claude/projects/{프로젝트키}/memory/MEMORY.md`
