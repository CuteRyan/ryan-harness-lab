# Checklist (통합 작업 체크리스트)

작업 전 체크리스트를 만들고, 완료 후 검증까지 하나의 흐름으로 관리하는 스킬.
코드 수정이든 문서 수정이든 하나의 `.checklist.md`로 통합 관리한다.

## Trigger
- 사용자가 작업을 지시할 때 (코드 수정, 문서 작성, 리팩터링 등)
- "체크리스트 만들어", "작업 시작", "이거 해줘" 등
- `/checklist` 직접 호출
- `/checklist 로그인 버그 수정` 처럼 작업명과 함께 호출

## 워크플로

### Phase 1: 체크리스트 생성
1. 작업 내용 분석 → 수정 대상이 코드인지 문서인지 자동 판단
2. 프로젝트 루트에 `.checklist.md` 생성
3. **주인님 승인** 후 다음 단계 진행

### Phase 2: 구현
- 체크리스트 항목을 하나씩 수행하며 `[x]` 체크
- 예상 외 이슈 발생 시 체크리스트에 항목 추가

### Phase 3: 검증 (구현 대조)
1. **항목별 1:1 대조** — 체크리스트 각 항목에 대해 실제 파일을 Read하여 반영 확인
2. **누락 탐지** — 체크리스트에 없지만 추가로 변경한 파일이 있으면 소급 추가
3. **코드**: ruff 린팅 + 관련 테스트 실행
4. **문서**: 연관 문서와 교차 검증 (용어/개념 일치, 링크 정상)
5. **사이드 이펙트** — 수정하지 않은 기존 기능이 깨지지 않았는지

### Phase 4: 더블 체크
1. **체크리스트 자체 검증** — 빠진 항목 없는가? 요구사항 다시 읽기
2. **실행 검증** — 체크만 하고 실제 반영 안 된 것은 없는가?
3. **일관성 검증** — 수정 내용이 다른 코드/문서와 여전히 일치하는가?

### Phase 5: 보고
주인님께 구조화된 리포트 제출. "완료했습니다"만 말하는 것은 금지.

필수 포함:
1. **완료 항목** — 무엇을 했는가 (파일명:라인 포함)
2. **검증 결과** — 린팅/테스트/교차검증 각각 통과/실패
3. **발견된 이슈** — 예상 외 문제 (없으면 "없음")
4. **미완료/보류** — 있으면 사유 (없으면 "없음")
5. **대조 결과** — N개 중 N개 완료, 불일치 N건

### Phase 6: 정리
- **주인님 승인 후** `.checklist.md` 삭제 — 자의적 삭제 금지
- 필요 시 HISTORY.md 업데이트

## `.checklist.md` 템플릿

```markdown
# 체크리스트: [작업명]

approved: true

## 작업 항목
- [ ] 항목 1
- [ ] 항목 2

## 수정 대상 파일
- `path/to/file` — 변경 내용

## 검증 항목
- [ ] 코드: 린팅 통과 / 문서: 연관 문서 교차 검증
- [ ] 기능 테스트 또는 링크 확인
- [ ] 기존 기능 영향 없음

## 더블 체크
- [ ] 체크리스트에 빠진 항목 없는가?
- [ ] 실제 반영 안 된 것은 없는가?
- [ ] 다른 코드/문서와 일관성 유지되는가?
```

## Rules
- **하나의 체크리스트** — 코드든 문서든 `.checklist.md` 하나로 관리
- **승인 마커 필수** — `approved: true` 또는 `- [x] 승인`
- **더블 체크 필수** — Phase 4 없이 완료 보고 금지
- **tiny edit 예외** — 3줄 이하 240자 이하 수정은 체크리스트 없이 진행 가능
- **백업은 doc-protection 훅이 담당** — 이 스킬에서 별도 백업하지 않음

## File Locations
- 체크리스트: `{프로젝트 루트}/.checklist.md`
- 참조: `docs/HISTORY.md`
