# Agent Team Manager

에이전트 팀을 구성, 저장, 불러오기, 실행, 관리하는 범용 스킬.

## 명령어

### 1. `/agent-team create [팀이름]`
새 에이전트 팀을 구성하고 저장한다.

**프로세스:**
1. 사용자에게 팀의 목적, 팀원 역할, 각 팀원의 지시사항을 물어본다
2. 팀 정의 파일을 **현재 프로젝트의 `.claude/teams/[팀이름]/`** 디렉토리에 저장한다
3. 팀 정의 파일 구조:
   ```
   .claude/teams/[팀이름]/
   ├── team.md          # 팀 개요, 목적, 실행 흐름
   └── members/
       ├── member-1.md   # 팀원 1 역할 및 지시사항
       ├── member-2.md   # 팀원 2 역할 및 지시사항
       └── ...
   ```

**team.md 형식:**
```markdown
# [팀이름]

## 목적
팀이 해결하려는 문제/목적

## 팀원 구성
| 이름 | 역할 | 설명 |
|------|------|------|

## 실행 흐름
1. 어떤 팀원이 먼저 실행되는지
2. 어떤 팀원이 병렬로 실행되는지
3. 어떤 팀원이 다른 팀원의 결과를 기다리는지

## 입력 형식
팀 실행 시 필요한 입력값

## 출력 형식
팀이 최종 생산하는 결과물의 형식
```

**member-N.md 형식:**
```markdown
# [팀원 이름]

## 역할
한 줄 요약

## 상세 지시사항
- 구체적으로 무엇을 해야 하는지
- 어떤 도구를 사용하는지
- 어떤 형식으로 결과를 내는지

## 사용 도구
- WebSearch, WebFetch, Bash 등

## 출력 형식
이 팀원이 내놓는 결과물의 형식
```

### 2. `/agent-team run [팀이름] [인자...]`
저장된 에이전트 팀을 불러와서 실행한다.

**프로세스:**
1. 현재 프로젝트의 `.claude/teams/[팀이름]/` 에서 팀 정의를 읽는다
2. team.md의 실행 흐름에 따라 팀원을 spawn한다
3. 병렬 실행 가능한 팀원은 동시에 spawn한다
4. 팀원 간 의존성이 있으면 순서대로 실행한다
5. 모든 팀원의 결과를 종합하여 최종 보고서를 생성한다
6. 보고서를 프로젝트의 `reports/` 디렉토리에 저장한다

**팀원 spawn 방식:**
- Claude Code의 Agent Teams 기능을 활용한다
- 각 팀원은 독립적인 teammate로 spawn된다
- teammate 간 직접 소통이 가능하다
- 공유 작업 목록을 통해 조율한다

### 3. `/agent-team list`
현재 프로젝트에 저장된 모든 에이전트 팀 목록을 보여준다.

**프로세스:**
1. 현재 프로젝트의 `.claude/teams/` 디렉토리를 탐색
2. 각 팀의 이름, 목적, 팀원 수를 표로 출력

### 4. `/agent-team show [팀이름]`
특정 팀의 상세 구성을 보여준다.

**프로세스:**
1. 해당 팀의 team.md와 각 member 파일을 읽어서 출력

### 5. `/agent-team edit [팀이름]`
기존 팀의 구성을 수정한다.

**프로세스:**
1. 현재 팀 구성을 보여주고
2. 사용자에게 수정할 부분을 물어보고
3. 해당 파일을 업데이트

### 6. `/agent-team delete [팀이름]`
저장된 팀을 삭제한다. 삭제 전 반드시 사용자에게 확인.

## 주의사항
- 팀 정의 파일은 항상 **현재 프로젝트** 내에 저장 (글로벌 아님)
- 팀 실행 시 결과 보고서도 **현재 프로젝트**의 `reports/`에 저장
- 팀원 spawn 시 충분한 컨텍스트(목적, 지시사항, 출력 형식)를 전달
- 실행 결과를 사용자에게 요약 브리핑 제공
