## 구현 과정에서 실측으로 드러난 이슈

### H1. PowerShell 5.1 한글 인코딩 (중대)
- **증상**: BOM 없는 UTF-8 PS1 파일을 PS 5.1이 CP949로 잘못 해석 → 한글 바이트가 `}` 같은 토큰으로 오인되어 파서 실패
- **해결**: 6개 PS1 모두 **UTF-8 BOM**(EF BB BF)으로 재저장
- **교훈**: PS 5.1 환경에서 한글 포함 PS1은 반드시 BOM 포함 저장. Write 도구는 기본 BOM 없음 → 작성 후 `System.Text.UTF8Encoding $true` 로 재저장 필요.

### H2. PowerShell 문자열 백틱 이스케이프
- **증상**: 프롬프트에 markdown 코드 마커 `` `...` `` 넣었더니 PowerShell이 백틱을 이스케이프 문자로 해석 → 문자열이 다음 `"` 까지 안 닫힘
- **해결**: 해당 라인은 single-quoted 문자열로 전환, 코드 마커를 `"`로 대체
