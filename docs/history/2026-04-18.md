### 7-6. Codex 쪽 5차 실측 점검 (커밋 후 추가 검증)
실제 `codex.cmd exec` 명령 실행 스모크 테스트로 3건 보강:
- **Phase 2 기본 예시 전환**: `--full-auto`(workspace-write) → `--sandbox read-only`가 기본. 리뷰 목적이 기본값. Gemini도 `--yolo` → `--approval-mode plan` 기본으로 동일 전환. 쓰기 필요 시에만 교체
- **UTF-8 가드 한계 명시**: PowerShell Constrained Language 모드에서 `[Console]::OutputEncoding` 설정이 `PropertySetterNotSupportedInConstrainedLanguage` 에러로 실패 (실측). Codex가 내부에서 명령별 UTF-8 prefix로 재처리하므로 최종 결과 정상. "가드 에러 무시 가능" + 출력 파일 저장 시 `Out-File -Encoding UTF8` 명시 권장 추가
- **`--search` 완전 예시 추가**: `codex.cmd --search exec --skip-git-repo-check --sandbox read-only ...` 루트 옵션 위치 확정. `exec --search`는 지원 안 됨
- **검증**: `--sandbox read-only`가 실제로 파괴 명령(`certutil -encodehex`) 차단 확인 (`rejected: blocked by policy`)

### 14-1. PowerShell 프로필 null 참조 스팸
- **증상**: 매 PS 명령마다 `Join-Path/Split-Path: 'Path' 매개 변수가 null` 에러 수십 회 반복
- **원인**: `Microsoft.PowerShell_profile.ps1`의 `Auto-ActivateVenv` 함수가 `$dir.Path`에 의존. `Get-Location`은 `.Path` 속성을 가진 PathInfo를 반환하지만 두 번째 반복의 `Get-Item $p`는 `.Path` 없는 DirectoryInfo를 반환(`.FullName`만 존재) → null 참조 연쇄
- **수정**: 문자열(`(Get-Location).Path`)로만 다루고 루프 전체에서 string 유지
- **추가 개선**: `.venv`를 `venv`보다 우선 탐색 (글로벌 `coding.md` 규칙 준수)
- **검증**: `powershell -NoProfile -Command ". <profile>"`로 에러 없이 로드 확인
- 변경: `C:/Users/rlgns/OneDrive/문서/WindowsPowerShell/Microsoft.PowerShell_profile.ps1`

### 14-3. VS Code 터미널 venv 활성화 명령어 가시 echo
- **증상**: 터미널 열 때마다 `(Set-ExecutionPolicy -Scope Process ...) ; (& ...\Activate.ps1)`가 프롬프트에 그대로 echo. 모든 프로젝트에서 반복
- **1차 진단 실패 + 학습**: ExecutionPolicy가 원인이라 가정하고 `Set-ExecutionPolicy -Scope CurrentUser RemoteSigned`를 제안. **확인 결과 CurrentUser 이미 `RemoteSigned`**. 즉 ExecutionPolicy와 무관. 실측 먼저라는 원칙 재확인.
- **실제 원인**: VS Code Python 확장이 `python.terminal.activateEnvironment` 기본값(true)으로 **ExecutionPolicy와 독립적으로** 매 터미널마다 activation wrapper 명령을 주입
- **수정 — `%APPDATA%/Code/User/settings.json`**:
  - `python.terminal.activateEnvironment: false`
  - `python-envs.terminal.autoActivationType: "off"` (신 Python Environments 확장 대응)
- **대체 메커니즘**: 14-1에서 고친 프로필 `Auto-ActivateVenv`가 조용히 활성화 담당 (echo 없음, `.venv`/`venv` 둘 다 지원)
