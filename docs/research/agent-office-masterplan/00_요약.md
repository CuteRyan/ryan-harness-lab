# Agent-office 마스터플랜 — 압축 요약

---

## Executive Summary (한 줄)

> 프로젝트마다 Opus PM을 두어 4가지 워커(인턴/회의실/외부CLI/파이프라인)를 동적으로 선택하고, /feedback이 단발 외부 검수하며, 모든 결정은 주인님이 최종 컨펌한다.

---

## 5층 위계 다이어그램

```
[1층] 주인님 (오너)              ← 최종 컨펌 / 검수
   │
[2층] 사장 (메인 Claude, Opus)   ← 코치 / spawn 대행 / 총괄
   ↕ 1:1 양방향 토론
[3층] 부장 (PM, Opus, 1인 팀)   ← 비판자 + 워커 추천
   ↓ 추천 → lead 대행 spawn
[4층] 워커 (4갈래 동적 선택)
   ① 인턴       ② 회의실      ③ 외부CLI    ④ 파이프라인
   Sub-agent    Agent Teams   Codex/Gemini  zircote 7패턴
   Sonnet       Sonnet        외부모델      Sonnet
   ↓ 완료
[5층] 검수 (/feedback)          ← 단발·ephemeral·외부CLI
   ↑
[1층] 주인님 최종 검수
```

---

## 4가지 워커 표

| 워커 | 언제 | 메커니즘 | 모델 | 특징 |
|------|------|---------|------|------|
| ① 인턴 | tool call ≤ 10, 단순 조회/탐색 | Agent tool 직접 호출 | Sonnet | 빠름·저비용·PM 게이트 생략 가능 |
| ② 회의실 | 복잡 협업, 다관점 병렬 | TeamCreate + preset | Sonnet | v2 스펙 Phase 0~8 |
| ③ 외부 CLI | 외부 검증, Echo chamber 회피 | /feedback orchestrate.ps1 | Codex/Gemini | 진짜 다른 모델 |
| ④ 파이프라인 | 단계 의존성, 고위험 변경 | TaskCreate + addBlockedBy 체인 | Sonnet | zircote 7패턴 |

## D-1~D-5 결정 표

| 결정 | 내용 | 비고 |
|------|------|------|
| **D-1** PM 메커니즘 | Agent Teams 1인 팀 + 비판자·선택자 두 역할. **PM은 추천만, spawn은 lead 대행** | issue#32731 확인 |
| **D-2** /feedback 통합 | 단발 유지 + 헬퍼 라이브러리 공유. 단발성 = 앵커링 회피 본질 | scripts/ 6개 공유 |
| **D-3** 워커 lifecycle | persistent(작업류) vs ephemeral(검증류) 분리. /feedback = ephemeral | TeammateIdle 훅 강제 가능 |
| **D-4** 모델 배분 | Opus: 사장·PM·/feedback 해석. Sonnet: ①②④ 워커. 외부CLI: ③+/feedback 검증 | 3개 출처 일치, 80% 비용 절감 |
| **D-5** 최종 결정권 | 주인님 = 오너. 모든 합의안에 컨펌 필수. 거부 시 다시 토론 | 글로벌 CLAUDE.md와 일관 |
