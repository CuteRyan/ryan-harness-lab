### 4.2 모델 선택 (출처: developers.openai.com/codex/models)

추천 모델 라인업:
- **`gpt-5.5`** — newest frontier (complex coding, computer use, knowledge work)
- `gpt-5.4` — flagship (강한 reasoning + agentic)
- `gpt-5.4-mini` — 빠르고 가벼움
- `gpt-5.3-codex` — coding 특화 (industry-leading)
- `gpt-5.3-codex-spark` — research preview, real-time iteration 최적화
- `gpt-5.2` — 기존 general-purpose

기본값: 명시 안 하면 `gpt-5.4` (5.5 unavailable 시) → **즉, `--model gpt-5.5` 명시 필요**

선택 방법:
- `codex -m gpt-5.5` (CLI 시작 플래그)
- `/model` 명령 (런타임 전환)
- `config.toml` 의 `model = "gpt-5.5"` (영구)

### 4.3 별도 제품 vs CLI 업데이트?

- **CLI 업데이트** — 별도 바이너리 아님. 기존 `codex` CLI 가 `--model gpt-5.5` 추가 지원
- 단 OpenAI 는 "GPT-5.2-Codex" 라는 **별도 코딩 특화 모델** (`gpt-5.3-codex`) 도 운영 중 → 이건 GPT-5.5 와는 다른 모델 군
- **결론**: 우리가 즉시 쓸 수 있는 것은 `codex --model gpt-5.5` (CLI 업데이트만 하면 됨)

## 5. 한국어 코드 리뷰 품질

### 5.1 직접 측정값 부재

- OpenAI 공식 발표·시스템 카드는 **한국어 특화 벤치마크 미공개**
- 검색 인덱스에 따르면 GPT-5.2 가 Korean Sator Square Test 에서 165.9/190 (Gemini 3 Pro 보다 +10) — GPT-5.2 기준이며 GPT-5.5 한국어 성능 직접 수치 없음
- ChatGPT 95+ 언어 지원 일반 보장은 있으나 코드 리뷰 한국어 품질은 별도 검증 필요

### 5.2 CodeRabbit 벤치마크 (영어 기준, 한국어 아님 주의) (출처: coderabbit.ai/blog/gpt-5-5-benchmark-results)

**Curated Review Benchmark**:
- Expected issue found: 79.2% (baseline 58.3%)
- Precision: 40.6% (baseline 27.9%)
- Comments: 75 (baseline 67)

**Large-Scale Real-World Review**:
- Expected issue found: 65.0% (baseline 55.0%)
- Precision: 13.2% (baseline 11.6%)
- Comments: 722 (baseline 558)

**정성**:
- "Scoped changes, preserved behavior, focused on actual failure mode"
- "Bias toward small workable changes instead of broad rewrites"
- 즉 GPT-5.5 는 **broad rewrite 욕심을 줄이고 핵심 결함만 짚는 성향** — Gemini 의 rubber-stamp 와 정확히 반대 방향

### 5.3 한국어 추정 (직접 측정 없음, 추론)

- GPT-5 계열은 다국어 자연 처리에서 Gemini 계열과 대등 또는 우위 (GPT-5.2 Korean Sator 결과 근거)
- 코드 리뷰의 핵심은 **코드 자체** + **간략한 자연어 의견** — 한국어 출력 품질은 자연어 부분만 영향
- 우리 `/feedback` 의 출력 양식 (`[태그] 제목` + 본문) 강제는 GPT-5.5 의 instruction following 능력 (CodeRabbit 정성평가 우수) 으로 안정적 처리 가능 추정
- **검증 필요**: 운영 투입 전 한국어 프롬프트 1~2회 dogfood 필수

## 7. 한글 경로 이슈 (codex-cwd 표준 패턴 영향)

### 7.1 우리 현행 패턴 (CLAUDE.md 명시)

```
$HOME\codex-cwd\<슬러그>.md  ← 영문 경로
codex.cmd exec --sandbox read-only "<프롬프트>"
--add-dir 최소 사용 (재귀 스캔 트리거)
```

배경: 한글 경로에서 CP949 깨짐 / UTF-8 재시도 정책 차단 / 재귀 스캔 폭주 (2026-04-19 실측 3종).

### 7.2 GPT-5.5 통합 후에도 동일 회피 필요한가?

**결론: 그렇다 — 동일 회피 패턴 유지 필요.**

근거:
1. **GitHub Issue #15158 (2026-03)**: Codex Desktop on Windows + Korean Unity C# files. UTF-8 검증 실패 시 silent UTF-8 변환 → 한글 깨짐. **"Closed as not planned"** — 픽스 의지 없음
2. **GitHub Issue #7290 (2025-11)**: Windows PowerShell 에서 비-라틴 문자 garbled. Root cause: `-NoProfile` 로 자식 프로세스가 system codepage (CP936/CP949 류) 로 회귀. Issue 는 닫혔으나 우리 환경에서는 여전히 재현 가능성 있음
3. **GitHub Issue #9671**: Windows 에서 Codex 가 cwd 무시하고 `C:\` 에서 시작
4. **OpenAI Community 1377898**: Windows username 에 특수문자 있으면 Codex 시작 실패

→ **GPT-5.5 출시 자체는 인코딩 레이어를 손대지 않음**. 위 이슈들은 모두 CLI 의 **Windows 프로세스 기동·파일 IO 인코딩** 문제이며 모델 교체와 무관. 따라서 **codex-cwd 영문 workdir 표준 패턴은 GPT-5.5 사용 시에도 그대로 유지** 필수.

추가 권고:
- Codex CLI v0.124.0 의 hooks stable 화로, 우리 `prepare-isolation.ps1` 같은 외부 스크립트 대신 hooks 로 격리 정책 정의 가능 → **장기 개선 후보** (별도 세션)
- `codex exec --json` (v0.125.0) 의 reasoning-token 보고를 활용하면 `/feedback` 의 비용 추적 정확도↑

- Codex 의 sandbox 격리는 잃지만 read-only 라는 코드 리뷰 성격상 위험도 낮음
- 마이그레이션 작업량: **중간** — `run-gemini.ps1` 을 `run-gpt5mini.ps1` 로 신설 + Validation Gate 정규식은 그대로 (`(?m)^\s*...\[태그\]`), `_encoding.ps1` dot-source 유지, 인증은 OPENAI_API_KEY 환경변수
- 예상 작업 시간: 2~3 시간 (구현 + 3 CLI 스모크 검증)

**옵션 3 (적극, 통합)**: Gemini 제거 + Codex 슬롯의 모델을 GPT-5.5 로 격상. 슬롯을 **2 CLI** (Claude Sub + Codex@5.5) 로 축소.
- 가격: Codex 슬롯만 2배. Gemini 운영비용 0
- 다양성↓ (Anthropic + OpenAI 만)
- 마이그레이션 작업량: **소** — `feedback` 스킬에서 Gemini 분기 삭제 + Codex `-m gpt-5.5` 명시
- **비추** — 3-way 다양성을 잃는 비용이 더 큼. 두 CLI 만으로는 cross-validation 약화

### 8.2 추천: **옵션 2 (GPT-5 mini API 직호출)**

이유:
1. **글로벌 CLAUDE.md 베이스라인 일치** — "최신 모델 기준" 원칙. GPT-5 mini 는 글로벌 추천
2. **한글 경로 우회 자체가 사라짐** — codex-cwd 표준 패턴은 Codex 슬롯에만 필요해짐 → 운영 단순화
3. **가격이 Gemini Free 손실을 상쇄** — Gemini Free 일 무료 호출 한도와 GPT-5 mini $0.25/$2.00 비교 시, 한 PR 리뷰 평균 5K input + 2K output = $0.0053 (약 7원). 일 5회 호출 시 월 $0.80 (약 1,100원) — 사실상 무시 가능
4. **rubber-stamp 회피 가능성** — OpenAI 모델은 OpenAI Codex 와 같은 조사 패밀리지만 mini 는 별도 학습. CodeRabbit 데이터 (precision 40.6%) 처럼 OpenAI 계열은 critique 성향 있음 (Gemini 3 의 rubber-stamp 와 다름)
5. **Codex 슬롯과 모델 분리 유지** — Codex 슬롯이 `gpt-5.4` 이면 mini 와 구분, `gpt-5.5` 로 격상해도 frontier vs mini 로 다양성 확보

### 8.3 추가 권고
