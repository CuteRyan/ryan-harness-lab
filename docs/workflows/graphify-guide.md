# Graphify 지식 그래프 관리 가이드

> rules/graphify.md의 상세 절차. 코드 프로젝트에서의 그래프 활용법.

---

## 적용 조건

- 프로젝트에 `.py`, `.js`, `.ts`, `.go`, `.java`, `.rs` 등 **코드 파일이 10개 이상**일 때만 적용
- 문서만 있는 프로젝트(docs/ 위주)에는 불필요

## 프로젝트 진입 시

1. `graphify-out/GRAPH_REPORT.md` 존재 여부 확인
2. **없으면**: 주인님에게 "Graphify 그래프가 없습니다. `/graphify .`으로 생성할까요?" 제안
3. **있으면**: `GRAPH_REPORT.md`를 프로젝트 구조 파악에 활용 (소스 직접 읽기 전에 먼저 참조)

## 코드 작업 중

1. `GRAPH_REPORT.md`가 있으면 함수/클래스 관계 파악 시 우선 참조
2. 파일 추가/삭제, 함수 시그니처 변경 등 **구조적 변경** 발생 시:
   - 작업 완료 후 주인님에게 "그래프 업데이트가 필요합니다. `/graphify . --update`할까요?" 제안
   - 강제 실행하지 않음 — 제안만
