# 위키(docs/) 관리 워크플로우

> rules/wiki.md의 상세 절차. index.md, log.md 포맷과 문서 생성/수정/점검 절차.

---

## 문서 생성 시 절차

1. **index.md 확인** — 같은 주제 기존 문서가 있는지. 있으면 새로 만들지 않고 업데이트
2. **관련 문서 탐색** — index.md에서 관련 카테고리 문서를 찾아 읽기
3. **교차 참조 삽입** — 새 문서 하단에 "관련 문서" 섹션
4. **기존 문서 역참조** — 관련 기존 문서에도 새 문서 링크 추가 (Edit으로)
5. **index.md 업데이트** — 적절한 카테고리에 새 문서 추가
6. **log.md 기록** — `## [YYYY-MM-DD] 동작 | 제목` 형식으로 추가

## 문서 수정 시 절차

1. **교차 참조 확인** — 수정 내용이 다른 문서의 설명과 충돌하지 않는지
2. **충돌 발견 시** — 관련 문서도 함께 수정하거나, 주인님에게 보고
3. **log.md 기록** — 수정 사항 기록

## index.md 포맷

```markdown
# 문서 인덱스

## 카테고리명
- [문서 제목](파일명.md) — 한 줄 설명
```

- 카테고리는 주제별로 묶음
- 각 항목은 `[제목](파일명.md) — 설명` 형식
- 문서 수가 늘면 카테고리 세분화

## log.md 포맷

```markdown
## [2026-04-12] create | 문서 제목
- 파일: `파일명.md`
- 교차 참조: `관련문서1.md`, `관련문서2.md` 업데이트
```

- append-only — 기존 항목 수정/삭제 금지
- 최신이 위로 (역순)

## 점검 (lint) — 주인님 요청 시

1. **고아 문서** — index.md에 없는 문서 탐지
2. **깨진 링크** — 교차 참조 링크가 실제 파일과 일치하는지
3. **구식 정보** — 작성일이 3개월 이상 지난 문서 목록
4. **중복 주제** — 비슷한 주제를 다루는 문서가 2개 이상인지
5. 결과를 주인님에게 리포트
