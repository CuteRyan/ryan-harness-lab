# 문서 작업 워크플로우

> rules/의 문서 관련 강제 규칙의 상세 절차. 신규 작성, 기존 수정, 설계 작업 모두 포함.

---

## 1. 신규 문서 작성

### 필수 선행 작업
1. **`.doc-checklist.md` 생성** — 프로젝트 루트
2. **목차(TOC) 먼저 작성** — 본문 전에 섹션 구조 잡고 주인님 승인
3. **체크리스트 생성** — 각 섹션별 "이것이 포함되어야 한다" 항목
4. **연관 문서 목록** — 참조하거나 영향을 주는 기존 문서 나열

### 체크리스트 필수 항목
- [ ] 목적/배경이 명확한가?
- [ ] 연관 문서와 용어/개념이 일치하는가?
- [ ] 미결정 사항이 명시되어 있는가?
- [ ] 다음 단계(Next Steps)가 있는가?

## 2. 기존 문서 수정

### 필수 선행 작업
1. **연관 문서 전부 읽기** — 참조하는 문서, 이 문서를 참조하는 문서 모두
2. **교차 검증 체크리스트 생성** — 문서 간 계약이 충돌하지 않는지 항목별 정리
3. **변경 영향 범위 파악** — 다른 문서에도 반영이 필요한지

### 교차 검증 체크리스트
- [ ] 동일 개념에 대한 설명이 문서 간 일치하는가?
- [ ] 데이터 흐름/상태 전이가 문서 간 모순 없는가?
- [ ] 스키마/필드명이 실제 코드/DB와 일치하는가?
- [ ] 한 문서에서 "불필요"라고 한 것을 다른 문서에서 "필수"로 두지 않았는가?

## 3. 코드/아키텍처 설계

### 필수 선행 작업
1. **체크리스트 생성** — 구현/설계 항목 빠짐없이 나열
2. **연관 코드/문서 확인** — 영향받는 파일, 모듈, 테이블 목록
3. **의존 관계 파악** — 순서가 중요한 작업은 의존 관계 명시

### 설계 체크리스트
- [ ] 입력/출력 계약이 명확한가?
- [ ] 기존 코드/스키마와 충돌하지 않는가?
- [ ] 에러/실패 케이스가 처리되어 있는가?
- [ ] 트랜잭션 범위가 명시되어 있는가? (DB 관련 시)

## 4. 작업 완료 조건 (더블 체크)

### Step 1: 체크리스트 전수 검사
- 항목 **전부 통과**해야 완료
- 교차 검증에서 충돌 → 관련 문서/코드 **모두 수정** 후 완료
- 미통과 항목 → 주인님에게 보고 후 판단 요청

### Step 2: 더블 체크
1. **체크리스트 자체 검증** — 빠진 항목이 없는가?
2. **실행 검증** — 체크만 하고 실제 반영 안 된 것은?
3. **일관성 검증** — 수정 내용이 다른 문서/코드와 일치하는가?
