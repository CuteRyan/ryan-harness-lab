# Claude 종합 — /feedback 커맨드 개정 dogfood 리뷰

## 메타 검증 — dogfood test 결과
- **프롬프트 길이 실측**: 600자 (3차 대비 1/10)
- **피드백 품질**: Codex 7개 지적(높음 3 + 중간 3 + 낮음 1) + 실측 검증(codex-cli 0.120.0, gemini 0.37.1), Gemini 4리스크 + 2아이디어. 품질 저하 없음 — 오히려 파일 직접 읽어 더 정확
- **결론**: **개정 방향 검증 성공**. 에이전트가 경로만으로 분석 가능. 장황함 감축으로 품질 향상

## Codex 요약 (실측 + 날카로움)
- **높음 3**: (1) Windows PowerShell에서 `codex`/`gemini` bare는 `.ps1` shim으로 실패 — `.cmd` 써야 함 / (2) `--full-auto`+`--yolo`는 쓰기 가능 → 리뷰는 읽기 전용 가드 필요 / (3) 출력 포맷이 "금지"와 "요청"으로 모순 — 1줄 고정 꼬리 필수
- **중간 3**: Codex `-C`/`--add-dir` 지침 부재, `--search` 위치 문제 + 검색 불가 시 한계 명시, 정보 부족 시 추측 금지 문구 필요
- **낮음 1**: "토큰 예산" → "프롬프트 예산"
- **핵심 제안**: **Phase 0 실행 프리플라이트** (CLI shim 확인 + 플래그 help + 경로 루트 산정 + 읽기/쓰기 모드 선택)
- **추천 꼬리 2줄**:
  ```
  제약: 읽기/분석만, 파일 수정·삭제·커밋 금지. 접근 실패나 맥락 부족은 추측 금지, 명시.
  출력: 중요도(치명/높음/중간/낮음), 파일:줄, 근거, 마지막 Top 3.
  ```

## Gemini 요약 (넓음)
- **긍정**: 방향성 탁월. "바로 적용 무방"
- **높음 2**: (1) 경로 해석 모호성(POSIX vs Windows) — 절대 경로 권장 / (2) 에이전트 읽기 실패 시 폴백 로직 부재 — "내용 포함 재시도" 규칙
- **중간 1**: `--include-directories` 공통 상위 자동 계산 지침 강화
- **낮음 1**: 피드백 인덱스 비대화 → 월별 아카이빙
- **추가 아이디어**: Pre-check(파일 존재 1차 확인), Diff 기반 리뷰(git diff 20~30줄)

## 공통 지적 (둘 다)
1. **프리플라이트/프리체크 필요** — Codex "Phase 0 프리플라이트" ≈ Gemini "Pre-check"
2. **읽기 실패 대응** — Codex "추측 금지" + Gemini "재시도 폴백" = 상호 보완
3. **경로 처리 명시성** — Codex "`-C`/`--add-dir` 지침" + Gemini "절대 경로 + Resolve-Path" = 같은 방향

## 주인님 추가 지적 (이번 대화)
- **한글 인코딩 이슈**: Codex 실행 시 PowerShell에서 한글이 깨지는 경우 있음
- **원인**: Windows 콘솔 기본 CP949, Codex가 한글 파일/경로 다룰 때 UTF-8 설정 안 하면 깨짐
- **가드**: Codex 호출 시 UTF-8 강제 prefix 또는 `chcp 65001`로 콘솔 코드페이지 전환 권장
