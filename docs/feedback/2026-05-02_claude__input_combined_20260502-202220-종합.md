## Top 3 반영 우선순위

1. **[높음] `/handoff` 양식 5종/6종 모순 정정** (지적 #2)
   - 즉시 가능. SKILL.md L66·L126 의 "5종 데이터 (마지막 상태 / 미완 / 다음 시작 / 미결 / 관련 파일)" → "6종 (... + 컨텍스트)" 또는 양식에서 컨텍스트를 선택 섹션으로 마킹.
   - **Why 시급**: 검증 기준 불일치 → 불완전 HANDOFF 통과 위험. 가장 단순한 수정인데 의미 무거움.

2. **[높음] `/todo` + `/handoff` 양식 drift 일괄 보강** (지적 #1, #3)
   - `/todo` 양식: priority enum (`high|normal|low`), `due:`, `blocked_by:`, `source:` 필드 공식 등재.
   - `/handoff` 양식: `## 🚨 사용자 결정 사항 (CRITICAL)` 섹션 + `Quick Start` 부절 (선택/필수 마킹) 추가.
   - **Why 시급**: 운영이 양식을 추월한 상태 = 양식이 문서로서 신뢰 잃음. 다른 프로젝트로 확산 시 이 drift 그대로 전파.

3. **[중간] `/handoff` ↔ `/checklist` 비대칭 해소 + `/checklist` Phase 6 트리거 명확화** (지적 #4, #5, #6)

   - `/checklist` Phase 6: "Phase 5 완료 보고 후 영구 기록 가치가 있으면 `/project-history update` 호출" 트리거 문장.
