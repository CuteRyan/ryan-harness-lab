# Claude 종합 — /feedback 스킬 비판 검토 강화안 메타 리뷰

> **메타 주의**: 이 종합문서는 제안된 A+B+C 포맷(Claude 검증 태깅 + 반박/유보 섹션 + 미확인 실측 TODO)을 **현장 적용**한 예시입니다. 형식 자체가 제안의 실효성 검증.

## 0. 세션 상황 (실측)
- **Gemini (gemini-cli 0.37.1)**: 2차 인라인 프롬프트에서 성공. 4.4KB 리뷰 생성. 품질 양호.
- **Codex (codex-cli 0.120.0, gpt-5.4)**: 3회 전 실패. 한글 경로 + PowerShell Constrained Language Mode + 정책 rejection 삼중 악재. 상세: `2026-04-19_codex_feedback스킬-비판검토-강화.md`
- **결과**: Codex 리뷰 부재 → 3자 크로스 리뷰 파이프라인이 **이미 1자 리뷰로 축소된 상태에서 종합**. 이 자체가 제안이 겨냥한 blind spot의 한 예 (단일 소스 의존).

## 1. Gemini 리뷰 요약 + Claude 태깅

### Gemini 보완 제안 (3개)
| # | Gemini 지적 | 중요도 | Claude 태깅 | 판단 근거 |

### Gemini 추가 리스크 (3개)
| # | Gemini 지적 | 중요도 | Claude 태깅 | 판단 근거 |
|---|---|---|---|---|
| R1 | **동반 환각 (Co-hallucination)** — 두 CLI가 같은 틀린 정보 줄 때 "공통 지적"으로 통과 | 치명 | ✅ 타당 | 내 원안 Phase 6의 "공통 지적 (✅만 반드시 반영)"도 여전히 공통≈타당 논리를 남겨둠. "공통일수록 더 엄격한 실측" 조항 필요. 단, 실측은 **코드/공식 문서 실제 확인**이어야지 제3 LLM 확인은 무의미 |
| R2 | **도구 제약 vs 프로젝트 버그 혼동** — 에이전트 도구 한계를 "파일 부재"로 오보고 | 높음 | ✅ 타당 + 실증 | **이번 세션 자체가 실증 사례** — Codex가 PowerShell policy rejection을 "읽기 실패"로 처리하고 무한 재시도. 🔍 미확인 태그 적극 사용 필수 |

## 4. 상충 지점

**Gemini vs Claude 원안**: 상충 없음. Gemini는 원안 방향 전면 지지 + 3가지 구조적 보강.

**Gemini vs Codex**: Codex 자체 리뷰 부재로 대조 불가 — **파이프라인 자체가 절반 가동 상태**.

## 5. 놓친 구조적 원인 (Claude 독자 발견)

주인님 점검 포인트 2번 (근본 해결책) — Gemini도 R1에서 완화책("공통일수록 엄격한 실측")은 줬으나 **근본 해결은 아님**. 내 원안도 마찬가지.

근본 해결 후보 (미검증, 제안만):
1. **비-LLM 검증 주입** — 린터/grep/공식 문서 URL fetch 같은 결정론적 수단으로 지적의 실체를 자동 확인. 5문항 중 1번(코드 일치)·3번(공식 문서)은 자동화 가능
2. **주인님 명시적 결재 단계** — 최종 반영 전 각 ✅에 대해 주인님 "승인/기각" 필수화. 지금은 묵시적 승인이 디폴트
3. **Codex 폴백 규약** — 3자 리뷰가 사실상 2자(이번 경우)가 될 때 "감축 경보" 발생 → Claude가 **더 엄격한** 태깅 기준 적용하거나 주인님께 재시도/대안 CLI 권고
