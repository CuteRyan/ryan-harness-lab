# Gemini 리뷰 — /feedback 커맨드 개정 (dogfood)

## 총평
"에이전트 자율성 극대화"와 "컨텍스트 압축" — **현대적 에이전트 오케스트레이션 방향에 매우 잘 부합**. 경로만 주면 에이전트가 `read_file`로 최신 상태를 직접 확인 → 정확도 향상. **바로 적용해도 무방**.

## 긍정 평가
1. **장황함 제거** — 포인터 방식으로 컨텍스트 오염 방지
2. **도구 활용 최적화** — `--include-directories`, `--full-auto`로 샌드박스 선제 해결
3. **역할 분담 구체화** — Codex(미시/사실) vs Gemini(거시/리스크) 상호 보완
4. **표준 포맷 강제** — 중요도 + 근거로 엔지니어링 리포트 가치 확보

## 리스크 및 보완 (우선순위)

### 1. 경로 해석 모호성 — 높음
- Windows에서 `~/.claude`(POSIX) vs `$env:USERPROFILE`(Windows) 해석 차이
- 에이전트가 샌드박스 내 `read_file` 시 상대 경로 꼬일 위험
- **제언**: **절대 경로(Absolute Path) 사용 권장** 또는 `Resolve-Path`로 Phase 1에서 경로 확정

### 2. 에이전트 '읽기 실패' 대응 부재 — 높음
- 네트워크/권한 이슈로 파일 못 읽으면 "분석 불가" 짧은 답변만 남기고 3,000자 소모
- **제언**: Phase 6 수집에서 결과물이 너무 짧거나(100자 미만) 에러 포함 시 **"내용 포함 재시도" 폴백 로직** 스킬 설명에 1줄 추가

### 3. Gemini `--include-directories` 확장성 — 중간
- 대상이 여러 드라이브/떨어진 경로면 누락 시 읽기 실패
- **제언**: "검토 대상의 **공통 상위 디렉토리 자동 계산**하여 인자 반영" 지침 강화

### 4. 장기 유지보수 — 피드백 인덱스 비대화 — 낮음
- `docs/feedback/index.md` 최상단 누적 → 시간 지날수록 비대
- **제언**: **월별 아카이빙** 또는 **최신 10개만 유지, 나머지 연도별 분리** 규칙 추가

## 추가 아이디어

### Pre-check
Phase 1에서 검토 대상 파일의 존재 1차 확인 → 불필요한 API 호출 비용 절감

### Diff 기반 리뷰
단순 경로 제공뿐 아니라 `git diff` 결과 짧은 요약을 프롬프트에 포함 → 에이전트가 "어느 부분이 바뀌었는지" 집중. **단 20~30라인 이내로 제한** (장황함 방지)

## 최종 평가
매우 훌륭한 개정. "피드백" 단어 중복 제거 같은 세밀한 네이밍 규칙에서 깊은 고민 느껴짐. 바로 적용 무방.
