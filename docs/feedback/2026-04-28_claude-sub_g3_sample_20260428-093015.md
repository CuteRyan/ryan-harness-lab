[치명] **SQL 인젝션** `g3_sample.py:11` — 문자열 연결로 쿼리 조립. `username = "' OR '1'='1"` 입력 시 전체 users 테이블 조회 가능. 파라미터 바인딩 필수: `cursor.execute("SELECT id, role FROM users WHERE name = ?", (username,))`

[치명] **하드코딩된 관리자 비밀번호** `g3_sample.py:6` — `ADMIN_PASSWORD = "admin1234"` 가 소스에 평문으로 존재. 리포 접근 권한 = 관리자 권한. `os.environ["ADMIN_PASSWORD"]` 또는 secrets manager로 이전 필수.

[높음] **비밀번호 평문 비교** `g3_sample.py:18` — `password == ADMIN_PASSWORD` 는 타이밍 공격(timing attack)에 취약. `hmac.compare_digest(password, ADMIN_PASSWORD)` 로 교체해야 함. (`secrets.compare_digest` 도 동일 효과)

[높음] **묵음 except** `g3_sample.py:24` — `except:` 는 `KeyboardInterrupt`, `SystemExit` 포함 모든 예외를 삼킴. DB 연결 오류·쿼리 오류를 구분할 수 없어 디버깅 불가. `except sqlite3.Error as e: logging.error(e)` 형태로 좁혀야 함.

[높음] **DB 연결 누수** `g3_sample.py:21–28` — `conn`을 `with sqlite3.connect(...)` 또는 `finally: conn.close()` 없이 열기만 함. 예외 발생 시(except 후 return None) 연결이 닫히지 않음.

[중간] **가변 기본 인자 (mutable default argument)** `g3_sample.py:16` — `history=[]` 는 함수 정의 시점에 한 번만 생성됨. 모든 호출이 같은 리스트를 공유해 호출 간 이력이 누적됨. `history=None` 후 `if history is None: history = []` 패턴 사용.

[중간] **일반 사용자 비밀번호 검증 없음** `g3_sample.py:27–28` — `find_user` 결과가 존재하면 비밀번호 확인 없이 로그인 성공 처리. `password` 인자가 일반 사용자 경로에서 전혀 사용되지 않음.

[낮음] **하드코딩된 DB 경로** `g3_sample.py:5` — `/var/db/users.db` 가 상수로 고정. 테스트·환경별 경로 교체 불가. 환경변수 또는 설정 주입 방식이 낫다.
