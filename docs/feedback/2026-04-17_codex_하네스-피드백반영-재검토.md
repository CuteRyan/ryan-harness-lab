# Codex 재검토: 하네스 피드백 반영 결과

> 모델: GPT-5.4 (OpenAI Codex v0.120.0) | 날짜: 2026-04-17

## 총평
수정은 일부 들어갔지만, "우선순위 1~6 완료"라고 보기에는 핵심 구멍이 남아 있다. 특히 MultiEdit는 settings에만 배선됐고 훅 본문이 MultiEdit를 무시해서 실제 보호가 동작하지 않는다. git 읽기 전용 허용도 현재 정규식 때문에 새 우회로가 생겼다.

## 주요 지적 (8개)

### 1. MultiEdit 배선은 됐지만 실제 훅이 전부 무시
- `doc-protection.sh:69` — Edit, Write, Bash만 처리, MultiEdit는 `*) exit 0`으로 빠짐
- `post-edit-verify.sh:25` — `Edit|Write`만 통과, MultiEdit 사후 검증 없음
- **결과**: MultiEdit로 문서 수정 시 백업도 안 되고, 검증도 안 됨. settings만 반쪽 수정

### 2. git 읽기 전용 화이트리스트가 redirection/체인 우회를 새로 만듦
- `doc-protection.sh:102`에서 git diff/show/status 등을 만나면 즉시 exit 0
- `git diff > docs/guide.md`, `git show HEAD:old.md > docs/guide.md` 등이 통과
- 전체 명령이 단일 git 조회 명령인지 끝까지 검증해야 함

### 3. git whitelist 자체도 읽기 전용이 아님
- branch, tag, remote, fetch를 통째로 허용
- `git branch -D`, `git tag -d`, `git remote remove` 등 상태 변경 가능
- 서브커맨드까지 제한 필요

### 4. git 파괴 명령 차단이 문서 확장자에 의존
- `git restore docs/`, `git checkout HEAD~1 -- docs/`, `git reset --hard`, `git clean -fd` 등 확장자 없이 통과
- 파괴적 git 명령은 확장자 유무 무관하게 기본 차단 필요

### 5. .checklist.md 인식은 됐지만 검증이 사실상 no-op
- post-edit-verify가 `- [x] path` 형태만 검증
- 실제 체크리스트 템플릿은 `- \`path/to/file\` — 변경 내용` 형태
- 파일명은 인식하지만 경로 존재 검증은 현재 포맷에서 안 돌아감

### 6. .backups/ 예외가 Windows 경로에서 깨짐
- `.backups/`만 매치 — Windows의 `.backups\guide.md`는 매치 안 됨
- `cp ... .backups\guide.md`가 mv/cp 차단에 걸릴 가능성

### 7. 배포 가드 CI 판정 취약점
- jq 결과가 비면 `${RUNNING:-0}`, `${NOT_SUCCESS:-0}` 때문에 성공으로 떨어짐
- `--limit 5` 때문에 워크플로 6개 이상이면 실패한 run이 목록 밖으로 밀림

### 8. ignored settings/settings.json이 옛 구조
- doc-doublecheck-guard, doc-checklist-guard, dev-checklist-guard가 남아있음
- 추적 대상은 아니지만 참조 설정으로 rules↔settings 충돌 재발 가능
