# /feedback 스킬 비판적 검토 기능 강화안

## 배경 — 주인님 지적

현재 `/feedback` 스킬 운영 결과, Claude가 Codex/Gemini의 피드백을
**비판적으로 검토하지 않고 단순 합성·요약**하는 패턴이 반복됨.

실제 증거 (2026-04-18_claude_feedback스킬-개정-종합.md):
- "상충 지점: 없음 → 역할 분담이 작동함" — 상충 없음을 "검증 성공"으로 해석 (사실은 검증 생략)
- "미반영 (의도): 없음 — 합리적인 지적 전부 즉시 반영 대상" — 거부/유보가 디폴트로 존재하지 않음
- "공통 지적 (반드시 반영)" — 두 CLI가 같은 말 하면 자동 타당으로 처리 (group-think)

## 진단 — 왜 비판적 검토가 안 되는가

### 구조적 원인 1: Phase 6 포맷이 합성 중심
현재 포맷:
```
Codex 요약 → Gemini 요약 → 공통 지적(반드시 반영) → 상충 → 우선순위 표
```
Claude 본인의 판단·반박·유보가 끼어들 섹션이 아예 없음.

### 구조적 원인 2: Phase 4에서 Claude 역할 미명시
현재:
- Codex: 날카롭게
- Gemini: 넓게

Claude 본인의 역할(게이트키퍼/검증자)은 선언되지 않음 →
기본값으로 "합성기/요약기"가 됨.

### 구조적 원인 3: 검증 체크리스트 부재
외부 CLI의 지적이 다음 케이스에 해당해도 필터링 단계가 없음:
- 환각 (존재하지 않는 플래그·파일·버전 언급)
- 과잉 일반화 (다른 프로젝트 기준 적용)
- 맥락 미스 (이미 해결된 문제 재지적)
- 공식 문서와 충돌

## 대책 — A+B+C 조합 제안

### A. Phase 4 재정의 — Claude 역할 명시

추가 문구:
> "Claude는 합성기가 아니라 **게이트키퍼**다.
> 두 CLI 의견을 무비판적으로 합치지 말 것.
> 실측·공식 문서·프로젝트 맥락으로 필터링한 뒤에만 '반영'으로 분류한다."

### B. Phase 6 포맷 재설계 — 검증 태그 의무화

각 지적마다 다음 4단계 중 하나 태깅:
- ✅ **타당** — 실측/문서로 확인됨
- ⚠️ **조건부** — 일부 맞으나 맥락/범위 제한 (어느 부분 제한인지 명시)
- ❌ **반박** — 환각/과잉/잘못됨 + 반박 근거
- 🔍 **미확인** — 판단 근거 부족, 실측 TODO

섹션 재편:
```
## Codex 요약 + Claude 태깅
## Gemini 요약 + Claude 태깅
## 공통 지적 (✅만 "반드시 반영")
## 반박/유보 항목 (❌ ⚠️ — 근거 필수)
## 미확인 → 실측 TODO
## 상충 지점 (판정 + 근거)
## 반영 우선순위 표
```

### C. Phase 6.5 — 검증 체크리스트 (자문 5문항)

각 지적 검토 시 Claude가 자문:
1. 실제 코드/문서와 일치? (실측)
2. 이 프로젝트 맥락에 적용? (일반론 vs 특수)
3. 공식 문서와 충돌?
4. 이미 해결/다른 곳 처리?
5. 환각 소지(존재 안 하는 플래그/파일)?

**타협**: 모든 지적에 5문항 전수 자문은 과할 수 있음 →
**"치명/높음/중간"만 5문항 전수, "낮음"은 간이 검증(1·5번만)**.

## 예상 트레이드오프

- **A만**: 선언 수준, 강제력 약함
- **B만**: 포맷만 바뀌고 검증 깊이 보장 안 됨
- **C만**: 체크리스트만 있고 출력에 안 보여 주인님이 검증 여부 확인 불가
- **A+B+C 조합**: 선언(A) + 강제 포맷(B) + 실행 절차(C) — 추천

## 리스크

1. **절차 과잉으로 오버헤드 증가** — 5문항 전수 자문이 피드백 1건당 수 분 추가될 수 있음
2. **태깅 자체가 가짜가 될 수 있음** — Claude가 "✅" 태그만 남발하면 무의미
3. **외부 CLI가 스킬 파일을 안 읽음** — Claude의 내부 절차이므로 CLI 프롬프트와는 무관 (이건 원래 그대로)
4. **"❌ 반박" 태그 남발 시 피드백 무용지물화** — 균형 필요
