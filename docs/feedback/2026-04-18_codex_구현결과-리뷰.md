### H1. deployment.md:35 순서 위험
`"이전 venv를 먼저 venv.old/로 리네이밍"` 순서는 auto-restart 걸리면 순간적으로 old path가 사라져 장애. **올바른 순서**: 새 `.venv` 생성 → smoke test → 서비스 중지 → unit/env 경로 반영 → `daemon-reload` → start → healthcheck.

### H2. B5 미완 + sync preview 0bytes
VS Code 문서상 `Accept Remote`는 로컬을 원격으로 덮고, `Accept Local`은 원격을 로컬로 덮음. 현재 로컬 sync preview는 0bytes, 과거 스냅샷도 빈 settings → B5에서 **반드시 `Accept Local`**. 실수로 `Accept Remote` 누르면 `%APPDATA%` settings 비게 됨(복구는 `_initial_backup` + `~/.claude/vscode/settings.json`으로 가능).

## 2. coding.md 품질 — 부분반영(중간)
- `workspaceSearchPaths`는 공식 Python Environments 설정이고 기본값 `["./**/.venv"]` 확인됨. OK
- **문제 coding.md:12 "이는 정상"**: 자동 기록 자체는 정상이나 **커밋 정상까지 보장하면 안 됨**. `python-envs.pythonProjects`는 `path`를 담아서 절대경로/홈/실행파일 경로가 들어갈 수 있음 → "기록될 수 있음. 커밋 전 절대경로·실행파일 경로 여부 검토"로 수정 권장
- **문제 coding.md:13 `.env override=True 필수`**: 운영 환경까지 일반화 위험. 서버에서는 시스템 env/secret manager가 `.env`보다 우선이어야 함 → "로컬 개발 기준" 또는 "프로젝트별 명시"로 좁혀야
