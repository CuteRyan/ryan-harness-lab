## Security & Code Review — `g3_sample.py`

---

### [CRITICAL]

[CRITICAL] **SQL Injection** — `g3_sample.py:11`
String concatenation builds the SQL query directly from unsanitized user input.
```python
query = "SELECT id, role FROM users WHERE name = '" + username + "'"
```
Payload `' OR '1'='1' --` dumps the entire users table. Fix: use parameterized queries (`cursor.execute("... WHERE name = ?", (username,))`).

[CRITICAL] **Hardcoded admin credential in source** — `g3_sample.py:6`
```python
ADMIN_PASSWORD = "admin1234"
```
Any reader of the source (developer, CI log, git history) gets permanent admin access. Fix: load from environment variable or secret manager; never store credentials in code.

[CRITICAL] **Non-admin login has no password check** — `g3_sample.py:27–28`
```python
if user:
    return {"id": user[0], "role": user[1]}
```
The `password` parameter is only compared against `ADMIN_PASSWORD`; for all other users the password is silently ignored. Any username that exists in the DB is accepted with any password.

---

### [HIGH]

[HIGH] **Admin access granted without validating user existence** — `g3_sample.py:18–19`
```python
if password == ADMIN_PASSWORD:
    return {"role": "admin"}
```
An attacker who knows the hardcoded password logs in as admin with a fabricated username that does not exist in the DB — no DB lookup is performed at all.

[HIGH] **Mutable default argument leaks cross-call state** — `g3_sample.py:16`
```python
def login(username, password, history=[]):
    history.append(username)
```
Python evaluates default arguments once at definition time. The same list persists across every call, accumulating all usernames in memory for the process lifetime — an information-disclosure bug and an unbounded memory leak. Fix: `history=None` then `if history is None: history = []`.

---

### [MEDIUM]

[MEDIUM] **Bare `except:` swallows all exceptions** — `g3_sample.py:24`
```python
except:
    return None
```
Catches `SystemExit`, `KeyboardInterrupt`, and any future programming error silently. Fix: `except Exception as e:` and at minimum log the error.

[MEDIUM] **DB connection never closed** — `g3_sample.py:21–29`
`conn = sqlite3.connect(DB_PATH)` is opened but there is no `finally: conn.close()` and no context manager (`with sqlite3.connect(...) as conn:`). Every successful login path leaks a file handle.

---

### [LOW]

[LOW] **Hardcoded filesystem path** — `g3_sample.py:5` (`DB_PATH = "/var/db/users.db"`) reduces portability and makes unit testing without mocking impossible; use an environment variable or config file instead.
