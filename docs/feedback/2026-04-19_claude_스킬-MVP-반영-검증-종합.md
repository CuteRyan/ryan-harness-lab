## 8. 실전 첫 적용 — MVP 스킬이 작동했는가?

### 작동한 것
- **판정 태그 강제** → 실제로 [반영] 5건 / [유보] 1건 / [반박] 4건으로 분류함. 무비판 합성 안 함
- **반박·유보 사유 빈 값 금지** → Gemini 환각 4건을 [반박] 태그 + 구체 근거로 처리. 이 섹션 없었으면 그냥 "공통 지적 = 반영" 해버렸을 것
- **치명/높음 [반영] 실측 증거 요구** → C1 근거로 "feedback.md:56 문구 + history:42-49 실패 기록" 명시
- **공통 지적 자동 반영 금지** → 공통 없음을 확인, 그룹싱크 없음

### 작동 안 한 것 / 개선 필요
- **C3** 지적대로 증거 범위가 좁음 — 명령 출력·공식 문서 URL도 허용해야
- **G8** 지적대로 1자 리뷰 상황 명시 규약 없음 — 오늘처럼 Gemini 환각 섞인 리뷰가 왔을 때 Claude가 "더 엄격히 검토해야 한다"는 자동 트리거가 없음

### 첫 dogfood 결론
**MVP 접근 유효**. 10줄 추가만으로 비판 검토 실제 강제됨. 단 3~5개 보강 항목 있음 (위 우선순위 표).
