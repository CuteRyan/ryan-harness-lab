# 배포 규칙

1. **배포 시 프로젝트 메모리의 `deployment.md`를 반드시 먼저 읽을 것** — SSH 접속정보, PEM 경로, 원격 경로, 서비스명이 전부 기록되어 있음. 메모리 없이 추측하거나 사용자에게 묻지 말 것
2. **글로벌 메모리 `aws-ec2-server.md`에 공통 서버 정보 있음** — 여러 프로젝트가 같은 서버를 공유하므로 서버 IP/PEM은 글로벌, 프로젝트별 경로/서비스는 프로젝트 메모리 참조
3. **배포 전 필수 체크리스트** (하나라도 빠지면 배포 금지):
   - 버전 번호 올리기 (pyproject.toml + index.html 스플래시 + 앱 정보 모달)
   - HISTORY.md 업데이트
   - 캐시버스팅 (`?v=` 파라미터 변경)
   - 린트 + 테스트 통과
   - git commit + push
4. **SCP 배포 후 반드시 서비스 재시작** — `sudo systemctl restart <서비스명>` 실행하고 running 상태 확인까지
5. **배포 완료 후 메모리 버전 갱신** — MEMORY.md의 버전 추적 항목 업데이트
